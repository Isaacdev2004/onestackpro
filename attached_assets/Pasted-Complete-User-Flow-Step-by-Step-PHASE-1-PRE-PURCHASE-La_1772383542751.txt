🔐 Security & Access Details
Encryption Key Behavior
Each user gets a unique encryption key

Key is generated immediately after Whop payment

Key is tied to their subscription status

If subscription lapses → key becomes invalid → DICloak access blocked

Key is stored securely, never exposed to other users

2FA Handling
Tool accounts have 2FA enabled (for admin security)

Users never see or enter 2FA codes

Platform automatically generates/grabs the current 2FA code when launching

Code is injected into the DICloak session automatically

User experience: click tool → tool opens logged in → no interruptions

Account Sharing Prevention
Encryption key is tied to device fingerprint (optional)

Concurrent session limits

IP monitoring for unusual activity

Mandatory 2FA on first login (prevents easy credential sharing)
